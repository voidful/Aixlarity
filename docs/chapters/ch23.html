第 23 章

插件系統

JSON 定義的工具基本架構，以及它目前在 codebase 中的真實狀態

實作狀態更新 (2026-04)

插件系統現已接入主執行路徑。App::exec 載入 PluginCatalog 並傳入 run_agent，與 built-in tools 和 MCP tools 一起合併進 live tool registry。此外，原有的指令注入漏洞已修復：不再使用 sh -c 展開模板，改用安全的 Command::new(program).args() 方式。Plugin 系統仍標為實驗中的 (experimental)，API 可能在未來版本變動。

插件定義格式

.aixlarity/plugins/lint.json
{
  "name": "lint_code",
  "description": "Run ESLint on a file",
  "parameters": {
    "type": "object",
    "properties": {
      "file": { "type": "string", "description": "File to lint" }
    },
    "required": ["file"]
  },
  "command": "npx eslint {{file}}",
  "max_output_bytes": 10240
}

這個模組已經完成的部分

  • 可以從全域與工作區插件目錄讀取 JSON 定義。
  • 可以把 {{param}} 模板展開成 shell command。
  • 可以包裝成實作 Tool trait 的動態工具元件。

已完成連接 (2026-04 更新)

  • App::exec 現在會載入 PluginCatalog（全域 + 工作區）並傳進 agent runtime。
  • agent.rs 的 run_agent 接收 plugin_tools 參數，透過 all_tools() 合併 built-in + plugin + MCP tools。
  • 指令執行改用 Command::new().args()，不再透過 shell 展開，防止命令注入。
