第 12 章

提示詞組裝引擎

所有的碎片如何組合成最終的 Prompt — prompt.rs

PromptAssembly 資料結構

prompt.rs — 核心結構 
pub struct PromptAssembly {
    pub final_prompt: String,       // 最終送出的文字
    pub provider: ProviderProfile,  // 使用的供應商
    pub sandbox: SandboxPolicy,    // 沙箱策略
    pub attachments: Vec<PromptAttachment>, // 附件
    pub pending_shells: Vec<String>, // 待審核 shell
}

組裝管線

1
命令解析 — 檢查輸入是否以 / 開頭，如果是則替換為命令定義的 prompt
2
檔案注入 — 展開 @{path} 語法，將檔案內容（限 400 行 / 32KB）或目錄列表（限 200 筆）注入
3
Shell 審核 — 提取 !{command} 區塊，標記為待審核
4
指令注入 — 載入 AGENTS.md / GCD.md 等指令檔
5
技能注入 — 如果有選中的技能，追加其內容
6
生成 PromptAssembly — 打包所有資料供 代理迴圈 使用

@{} 附件語法

語法效果限制
@{file.rs}注入檔案內容400 行 / 32KB
@{src/}注入目錄列表200 筆
@{image.png}標記為二進位參考不展開內容
