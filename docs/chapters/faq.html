名詞速查

8 個最容易搞混的概念

第一次接觸 AI 助手時，不需要一次把所有名詞都背下來。這裡幫你把最常搞混的幾組概念分清楚。

讀完這章大概需要 3 分鐘

目標很簡單：把最常混在一起的詞分開。知道每個詞管哪一段就好，不用記得很精確。後面讀到具體章節時，自然會越來越熟。

問題 1：模型、Provider、Protocol 差在哪？

名詞 白話解釋 舉例
模型（Model） 真正負責思考和生成文字的 AI 引擎 GPT-4o、Gemini 2.5 Pro、Claude 4 Sonnet
Provider 把模型提供給你使用的「服務設定」 同一個 GPT 模型，可以用不同的 API key、不同的伺服器網址
Protocol（協定） 你和 Provider 溝通時用的「API 格式」 OpenAI Chat 格式、Gemini 格式、Anthropic 格式（各家格式不同！）

簡單記法：模型是引擎、Provider 是加油站的設定、Protocol 是加油孔的規格。

問題 2：Prompt、Command、Skill 差在哪？

名詞 白話解釋 它負責什麼
Prompt（提示詞） 最後真正送給 AI 看的那份完整文字 把你的輸入、專案規則、上下文組合成一份任務說明
Command（命令） 某類任務的模板 像 /review、/refactor 這種預設的任務格式
Skill（技能） 一套方法論的知識包 例如「code review 要檢查哪些面向」「Rust 測試要注意什麼」
簡單記法 
Prompt = 最後送進 AI 的那份完整文字
Command = 某類任務要怎麼展開（模板）
Skill = 用哪一套方法去做（知識包）

問題 3：Tool、Shell、Patch 差在哪？

這三個很容易搞混，但其實它們不在同一個層次：

名詞 白話解釋 舉例
Tool（工具） AI 能呼叫的任何一種能力 read_file、search_files、shell、apply_patch 都算工具
Shell 工具的其中一種——執行終端機指令 跑 cargo test、git status、npm lint
Patch 工具的另一種——用結構化方式修改檔案 不是跑指令，而是直接套用一段程式碼變更

所以當文件說「AI 會用工具」，不一定代表它在跑 shell。讀檔、搜尋、寫檔、套 patch 都算工具，但它們的風險等級不同。

問題 4：Trust、Permission、Sandbox 差在哪？

這三個都和安全有關，但管的是三個不同的問題：

名詞 它在問什麼 白話解釋
Trust（信任） 這個專案的設定檔可以信嗎？ 決定專案自帶的 AGENTS.md、commands、skills 要不要載入
Permission（權限） 這個操作要不要先問人？ 像 suggest（只建議）、auto-edit（自動改）、full-auto（全自動）
Sandbox（沙盒） 就算允許做，實際能做到什麼程度？ 像 read-only（唯讀）、workspace-write（只能改工作區）、container（在容器裡跑）
舉例：AI 想跑 cargo test，三層各管什麼 
AI 想跑：cargo test

Trust（信任）：
  這個專案的設定檔可以信嗎？設定裡的規則要不要載入？

Permission（權限）：
  這次跑 shell 要不要先問你？

Sandbox（沙盒）：
  就算允許跑，是在你的電腦上直接跑、還是進 Docker 容器裡跑？

問題 5：Session、Checkpoint、Memory 差在哪？

名詞 白話解釋 用途
Session（工作記錄） 一次工作過程的完整記錄 讓你可以回放、續接、分叉
Checkpoint（快照） 某一刻的提示詞和執行狀態快照 方便回頭看「送進 AI 之前長什麼樣」
Memory（記憶） 跨多次工作都保留的長期筆記 記住專案的重要資訊，下次工作時自動帶入

最簡單的理解：Session 像這次工作的資料夾、Checkpoint 像其中某一刻的截圖、Memory 像一本長期筆記。

問題 6：Plan 模式和 Live 模式差在哪？

模式 白話解釋 什麼時候用
Plan（規劃模式） AI 只會產生計畫書，不會真的動手 想先看看 AI 打算怎麼做，再決定要不要讓它做
Live（執行模式） AI 會真的呼叫工具、修改檔案 確認方向沒問題後，讓 AI 實際動手

問題 7：Plugin、MCP、內建工具差在哪？

類型 白話解釋 為什麼要知道
內建工具（Built-in） 系統自帶的、最穩定的工具 大部分安全規則都是以這些工具為準來設計的
Plugin（插件） 你自己加的本地擴充工具 可以用，但不一定完全享有內建工具的安全保護
MCP（外部工具協定） 透過標準協定連接外部工具伺服器 不只是多一個指令，而是多了一層網路連線和協定

問題 8：為什麼文件裡都標 Implemented？

因為每個功能都有實際的程式碼支撐和自動化測試。這個標記的意思是：這不是只寫在規劃書上的功能，而是你可以在原始碼裡追查到的實作。
