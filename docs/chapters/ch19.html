外殼工程 原理總結

從 GCD 提煉的 AI AI 助手 設計原理

十大設計原理

1

分層安全 (Layered Security)

Trust → Sandbox → Permission 三重閘門，永遠不把安全決策交給 LLM。

2

最小權限 (Least Privilege)

預設 auto-edit + workspace-write，只在明確要求時才提升權限。

3

供應商無關 (Vendor Agnostic)

透過統一的 Provider 介面，將 LLM 視為可替換的元件。

4

確定性優先 (Determinism First)

所有非 LLM 的邏輯（工具執行、路徑解析、權限檢查）都是完全確定性的。

5

可觀測性 (Observability)

Token 追蹤、事件日誌、Session 回放讓每次 Agent 執行都可審計。

6

優雅降級 (Graceful Degradation)

Provider Fallback、Context 壓縮、輸出截斷 — 永遠不讓 Agent 完全失敗。

7

配置即代碼 (Config as Code)

AGENTS.md、TOML 命令、SKILL.md — 所有配置都可版本控制。

8

漸進式自動化 (Progressive Automation)

suggest → auto-edit → full-auto，讓使用者按自己的信任節奏逐步放手。

9

會話即歷史 (Session as History)

每次互動自動持久化，支援 Fork/Resume/Replay。不浪費任何一次 LLM 呼叫。

10

工具組合性 (Tool Composability)

內建工具 + MCP + 插件，三層可組合的工具架構讓擴充無上限。
