第 06 章

沙箱策略 (Sandbox Policy)

限制 Agent 的檔案系統與 Shell 存取範圍

四級沙箱模型

靈感源自 OpenAI Codex 的三級沙箱，Aixlarity 擴展為四級：

等級讀取寫入Shell隔離適用場景
off任意任意不限無完全信任環境
read-only工作區禁止唯讀弱程式碼審查
workspace-write工作區工作區限工作區中日常開發（預設）
container掛載掛載容器內強不信任的 repo

沙箱如何影響工具

每個內建工具在執行前都會檢查 ToolContext 中的 SandboxPolicy：

  • read_file — 檢查路徑是否在允許的讀取範圍內
  • write_file — 檢查路徑是否在允許的寫入範圍內
  • shell — 在 container 模式下，命令在 Docker/Podman 容器內執行
  • list_dir — 受讀取範圍限制
  • search_files — 受讀取範圍限制
容器沙箱

Container 模式使用 Docker 或 Podman 將工作區掛載到容器中。Agent 的所有檔案操作和 shell 命令都在容器內執行，主機系統完全隔離。詳見第 21 章。
