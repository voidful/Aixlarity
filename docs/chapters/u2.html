互動模式與基本操作

掌握 Aixlarity 的兩種工作模式與核心操作

兩種工作模式

1

REPL 互動模式

直接輸入 aixlarity 啟動。支援 rustyline 行編輯、歷史記錄、Ctrl+R 搜尋。適合探索式開發與多輪對話。

2

單次執行模式

使用 aixlarity exec "任務"。適合 CI/CD 流程、腳本自動化、一次性任務。支援 --json / --jsonl 輸出。

核心 CLI 命令

命令說明範例
aixlarity啟動 REPLaixlarity
aixlarity exec執行單次任務aixlarity exec "寫測試"
aixlarity overview檢視工作區概覽aixlarity overview --json
aixlarity providers管理 LLM 供應商aixlarity providers doctor
aixlarity trust管理信任規則aixlarity trust status
aixlarity sessions管理歷史會話aixlarity sessions list
aixlarity commands重載自訂命令aixlarity commands reload

exec 常用參數

參數說明預設值
--provider <id>指定 LLM 供應商自動偵測
--sandbox <level>沙箱隔離等級workspace-write
--permission <level>權限等級auto-edit
--git執行後自動 commit關閉
--plan輸出 execution plan 與 prompt 預覽，不直加入入 live 代理迴圈關閉
--resume <id>恢復舊會話—
--fork <id>複製舊會話繼續—
--checkpoint儲存提示詞快照關閉
--no-stream禁用串流輸出開啟
--no-session不持久化會話持久化
關於 --plan

依目前 App::exec 的實作，--plan 會建立計畫型輸出、可選 checkpoint 與 session 記錄，但不會自動進到後續 live execution。文件如果寫成「先規劃再執行」，容易讓人以為它會在同一次命令內完成兩階段工作。

權限模式詳解

suggest

建議模式

所有寫入操作需人工確認。最安全，適合不熟悉的 codebase。

auto-edit

自動編輯

檔案寫入自動執行，shell 和 patch 需確認。預設模式，平衡效率與安全。

full-auto

完全自動

所有工具自動執行。最快速，僅在信任環境使用。

提示詞增強語法

Aixlarity 支援特殊語法在提示詞中注入上下文：

prompt examples 
# 注入檔案內容
aixlarity exec "分析 @{src/main.rs} 的效能問題"

# 注入目錄結構
aixlarity exec "基於 @{src/} 目錄設計新模組"

# 使用自訂命令
aixlarity exec "/review src/auth.rs"

# Shell 區塊（需審核）
aixlarity exec "依據 !{cargo test} 的結果修復錯誤"
