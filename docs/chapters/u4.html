專案指令 (AGENTS.md)

在專案根目錄放置 AGENTS.md，GCD 會自動注入到 System Prompt：

AGENTS.md 
# 專案規範

## 程式碼風格
- 使用 4 空格縮排
- 函數名稱採用 snake_case
- 所有公開 API 都要有文件註解

## 測試要求
- 每個新功能都要有對應的單元測試
- 使用 `cargo test` 驗證

## 架構決策
- 避免使用 unwrap()，改用 ? 運算子
- 錯誤類型使用 thiserror 定義
多指令檔案

GCD 同時支援 GCD.md、GEMINI.md、CLAUDE.md 等供應商特定指令檔。當使用特定供應商時，會額外載入對應的指令。

自訂命令

在 .gcd/commands/ 目錄放置 TOML 檔案，定義可複用的命令：

.gcd/commands/review.toml 
description = "程式碼審查"
prompt = """
請對以下程式碼進行詳細審查，重點關注：
1. 安全性漏洞
2. 效能瓶頸
3. 錯誤處理完整性
4. 程式碼可讀性
5. 是否符合 AGENTS.md 中的規範

請用表格呈現發現的問題及建議修復方案。
"""

使用方式：gcd exec "/review src/auth.rs"

技能系統 (Skills)

在 .gcd/skills/ 目錄放置 Markdown 檔案，作為可選注入的專業知識：

.gcd/skills/rust-optimization/SKILL.md 
Rust 效能最佳化專家

# 最佳化策略
1. 避免不必要的 clone()，改用借用
2. 使用 Cow<str> 延遲分配
3. 優先使用 iterator 而非 collect()
4. 考慮使用 SmallVec 減少堆分配
5. Profile 後再最佳化，避免過早最佳化

MCP 伺服器

在 .gcd/mcp.json 中配置 MCP 伺服器，擴充 Agent 工具能力：

.gcd/mcp.json 
{
  "servers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"],
      "env": {}
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "${GITHUB_TOKEN}" }
    }
  }
}
