工作區發現機制

Aixlarity 如何找到你的專案根目錄 — workspace.rs

偵測策略

Aixlarity 從當前目錄開始往上搜尋，尋找以下標記檔案來識別專案根目錄：

1
.aixlarity/ — Aixlarity 專屬配置目錄（最高優先）
2
.git/ — Git 倉庫根目錄
3
Cargo.toml — Rust 專案根
4
package.json — Node.js 專案根
5
如果都找不到，使用當前目錄作為工作區

原始碼解析

crates/aixlarity-core/src/workspace.rs 
pub fn discover(start: &Path) -> PathBuf {
    let markers = [".aixlarity", ".git", "Cargo.toml", "package.json"];
    let mut dir = start.to_path_buf();
    loop {
        for m in &markers {
            if dir.join(m).exists() {
                return dir;
            }
        }
        if !dir.pop() { break; }
    }
    start.to_path_buf() // fallback
}
為什麼重要？

工作區根目錄決定了 Sandbox 的讀寫範圍、Trust 規則的評估、AGENTS.md 的載入位置、Session 的歸屬。它是整個安全設計的起點。
