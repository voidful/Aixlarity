專案概觀與設計哲學

Aixlarity 為何存在，以及它的設計核心信念

為什麼要重造輪子？

2025-2026 年，AI coding agent 生態已經出現 Claude Code、Gemini CLI、OpenAI Codex 等成熟 CLI 產品，也開始走向更完整的 IDE workbench。Aixlarity 並非要取代它們，而是要把可觀察、可教學、可驗證的 harness 設計重新整理成開源版本：

  • 被單一供應商綁住：Claude Code 只能用 Anthropic、Gemini CLI 只能用 Google、Codex 只能用 OpenAI。Aixlarity 讓你自由切換。
  • 啟動開銷：Node.js/TypeScript 實作的 Agent 啟動需要數百毫秒。Rust 原生二進位啟動僅需幾毫秒。
  • 記憶體效率：Rust 的零成本抽象讓 Aixlarity 在長時間 session 中保持低記憶體佔用。
  • 安全保證：Rust 的型別系統從編譯期就排除大量 執行時 錯誤。

五大設計原則

1

安全第一 (Security First)

每個破壞性操作都經過 Trust → Sandbox → Permission 三重閘門。不信任就不執行。靈感來自 Claude Code 的 Trust 系統與 Codex 的 Sandbox 分級。

2

供應商無關 (Vendor Agnostic)

LLM 供應商是可插拔的。今天用 Gemini，明天切 Claude，後天換 OpenAI。程式碼不需要任何修改。

3

會話即歷史 (Session as History)

每次互動都自動持久化。任何 session 都可以 resume、fork、replay。靈感來自 Claude Code 的 Session 分支機制。

4

漸進式信任 (Progressive Trust)

從最保守的 suggest 模式開始，隨著信任建立逐步放寬到 full-auto。靈感來自 Codex 的 Permission 模型。

5

可觀測性 (Observability)

Token 用量、工具呼叫、API 計時、事件日誌 — 一切操作都可追蹤與回放。

名稱由來

Aixlarity 來自 AI + clarity：用清楚、可追溯的方式理解 AI coding agent 的外殼。
它不是某個產品名稱的拼字遊戲，而是一個把 runtime、IDE、證據與教學放在一起的開源 harness project。
