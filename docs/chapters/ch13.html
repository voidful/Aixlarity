關鍵程式碼路徑

agent.rs — 簡化的核心迴圈 
pub async fn run(&self, opts: AgentRunOptions) -> Result<AgentResult> {
    let mut messages = build_initial_messages(&opts);
    let mut turn = 0;

    loop {
        turn += 1;
        if turn > MAX_TURNS { break; }

        // 呼叫 LLM (含 Streaming + Fallback)
        let response = call_provider(&messages).await?;

        // 提取工具呼叫
        let tool_calls = extract_tool_calls(&response);
        if tool_calls.is_empty() { break; }

        // 逐一執行工具
        for call in &tool_calls {
            if needs_permission(call, &opts.permission) {
                prompt_user(call).await?;
            }
            let result = execute_tool(call).await?;
            messages.push(tool_result_message(result));
        }

        // Context Window 管理
        if exceeds_budget(&messages) {
            messages = compact_history(messages);
        }
    }

    Ok(build_result(messages, token_usage))
}

Provider Fallback 機制

當 LLM API 返回 429 (Rate Limit)、500 (Server Error) 或連線逾時時，Agent 會自動嘗試備用供應商：

主要供應商 失敗
→
等待 + 重試
→
切換備用供應商
→
繼續 代理迴圈

Context 壓縮策略

當訊息歷史超過 token 預算時，runtime_support.rs 會保留第一則訊息與最近尾段，再把中段對話濃縮成摘要。這裡保留的是「目前 messages 陣列中的第一則訊息」，不是額外獨立的 system prompt 物件：

  1. 保留第一則訊息
  2. 保留最近尾端的幾則對話
  3. 將中段 assistant / tool 互動改寫為摘要
  4. 用新的 user-style summary message 取代中段歷史
