14 步執行管線全景

從輸入到輸出的完整旅程

完整管線

當使用者執行 aixlarity exec "重構 auth 模組" 時，以下 14 個步驟依序發生：

1
CLI 解析 — Clap 解析命令列參數，提取 prompt、flags、options
2
工作區發現 — 從當前目錄向上搜尋 .git / Cargo.toml 等標記
3
信任評估 — 檢查 trusted-folders.txt，決定信任等級
4
供應商解析 — 按優先順序找到可用的 LLM 供應商
5
指令載入 — 讀取 AGENTS.md / AIXLARITY.md（如果信任允許）
6
命令/技能載入 — 掃描 .aixlarity/commands/ 和 .aixlarity/skills/
7
MCP 連線 — 啟動 MCP 伺服器子程序，發現可用工具
8
提示詞組裝 — 展開 @{}、解析 /command、注入指令與技能
9
工具註冊 — 合併內建 + MCP + 插件工具為統一註冊表
10
Agent 迴圈 — 核心 10 輪迭代（LLM ↔ Tool 來回）
11
Session 持久化 — 將整輪對話寫入 ~/.aixlarity/sessions/
12
Memory 更新 — 如果有新知識，更新 MEMORY.md
13
Git Commit — 如果 --git 旗標開啟，stage + commit 變更
14
輸出渲染 — 將結果以 text / JSON / JSONL 格式輸出
